🔧 Test Fix Salvataggio Scenario

Descrizione del Bug Risolto

Problema: Quando si modificava uno scenario, le modifiche venivano salvate su uno scenario diverso.

Causa: La variabile this.currentScenario poteva essere sovrascritta durante la navigazione, causando il salvataggio sull'ID sbagliato.

Soluzione: L'ID dello scenario viene ora salvato direttamente nel form come data-scenario-id, garantendo che il salvataggio avvenga sempre sullo scenario corretto.

Test della Correzione

Questo test verifica che il fix funzioni correttamente:

Come Testare Manualmente

  1. Apri l'applicazione principale
  2. Crea o apri uno scenario (es. "Scenario A")
  3. Modifica alcuni campi (nome, destinazione, spese)
  4. Salva lo scenario
  5. Verifica che le modifiche siano state salvate sullo scenario corretto
  6. Ripeti con altri scenari per confermare
✅ Comportamento Atteso:
  • Le modifiche vengono salvate sempre sullo scenario che hai aperto
  • Non vengono più sovrascritti scenari diversi
  • Il messaggio di conferma mostra il nome corretto dello scenario

Modifiche Apportate

1. editScenario(id)
   - Aggiunto: form.dataset.scenarioId = scenario.id
   - Salva l'ID nel form quando si modifica uno scenario

2. createNewScenario()
   - Aggiunto: delete form.dataset.scenarioId
   - Rimuove l'ID dal form per nuovi scenari

3. viewScenario(id)
   - Aggiunto: form.dataset.scenarioId = scenario.id
   - Salva l'ID anche in modalità visualizzazione

4. saveScenario()
   - Modificato: Usa form.dataset.scenarioId invece di this.currentScenario.id
   - Garantisce il salvataggio sullo scenario corretto
   - Aggiunto logging per debug
